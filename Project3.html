Project 3: CapitalOne Financial Hack

What I learned

My team and I were planning on doing a data related challenge for Hack OHI/O. Unfourtunately, the organization running the challenge had difficulty getting their servers working properly. We had to leave our comfort zone: working with data. We shifted our plan of action. We were going to learn something completely new on the fly. None of us had experience in web development, but we decided to make a chrome extension for the CapitalOne financial hack challenge. I learned about the power of JavaScript in web development. The manifest for the extension was entirely JavaScript code. We only used HTML and CSS to change the extension's display. JavaScript was a hard language to learn informally. It's pretty complex, and it took me a while to make a simple spending checker. I gained a lot of respect for web developers. If I have time, I may try to delve deeper into JS.

Improvements

  • Increasing the visual appeal of our chrome extension
  • Merging webscraped Amazon data with the spending checker
  • Making price checker automated instead of manual (a-la Honey)
  • Making the websites that link to the extension more detailed
